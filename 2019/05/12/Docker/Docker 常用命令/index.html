Docker 常用命令

  3. 标记镜像

  1. 推送镜像

  1. 可以用prune命令删除不用的资源，比如：

  2. 显示所有的容器，过滤出Exited状态的容器，取出这些容器的ID
    sudo docker ps -a|grep Exited|awk '{print $1}'

  3. 查询所有的容器，过滤出Exited状态的容器，列出容器ID，删除这些容器
    sudo docker rm $(docker ps -a|grep Exited|awk '{print $1}')
    或者
    sudo docker rm $(sudo docker ps -qf status=exited)

  4. 删除所有未运行的容器（已经运行的删除不了，未运行的就一起被删除了）
    sudo docker rm $(sudo docker ps -a -q)

  5. 删除孤立的容器(Docker 1.13版本以后)
    sudo docker container prune

  6. 命令查看磁盘使用情况
    du -hs /var/lib/docker/

  7. 查看Docker的磁盘使用情况
    docker system df

  8. 清理磁盘，删除关闭的容器、无用的数据卷和网络，以及dangling镜像(即无tag的镜像)
    docker system prune

  9. 迁移/var/lib/docker目录下面的文件到/apps/data/docker
    rsync -avz /var/lib/docker /apps/data/docker/

  10. 删除 docker0 虚拟网卡

  11. 删除桥接网络flannel.1
    ifconfig flannel.1 down && ip link del flannel.1

  12. 清理iptables
    iptables -F && iptables -t nat -F && iptables -t mangle -F && iptables -t raw -F && iptables -X && iptables -A FORWARD -j ACCEPT

  13. 重建docker0网络

经过验证发现，当我们使用iptables -t nat -F清理iptables或其他误操作破坏了Docker0网络，可以采用启动Docker的方式自动创建docker0网络。
systemctl stop docker && ifconfig docker0 down && brctl delbr docker0 && systemctl start docker

  1. 运行完命令及删除容器
    docker run -it --rm maven:3.6.1-jdk-7 mvn version

  2. 特权模式运行容器
    docker run -it --privileged ubuntu bash
